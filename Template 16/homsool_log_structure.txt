======================================================================
  [SoolPair] 나만의 홈술로그 (homsool_log.html) 코드 구조 및 아키텍처 분석서
======================================================================

본 문서는 대학생 및 사회초년생을 위한 스마트 안주 페어링 플랫폼인 SoolPair의 신규 기능인 
'나만의 홈술로그(homsool_log.html)' 페이지의 코드 동작 방식과 전체 구조를 타인에게 쉽게 
설명할 수 있도록 분석한 자료입니다.

이 페이지는 프론트엔드 HTML5, Vanilla CSS, 그리고 Vanilla JavaScript만을 활용하여 구현된 
완전한 단일 페이지 애플리케이션(SPA) 스타일의 인터랙티브 컴포넌트입니다.

----------------------------------------------------------------------
1. 페이지 개요 (Summary)
----------------------------------------------------------------------
- 역할: 사용자가 달력에 스탬프를 찍듯이 매일 마신 술(위스키, 와인, 리큐르)을 기록하는 다이어리.
- 핵심 UX:
  1) 도장 툴바에서 원하는 주종(위스키/와인/리큐르)을 선택합니다.
  2) 1일부터 31일까지 배치된 달력 카드를 클릭하여 스탬프를 찍거나 지웁니다.
  3) 스탬프의 변화에 따라 우측의 '주종 소비 비율 차트'와 '홈술 건강 관리 가이드'가 
     어떠한 서버 요청이나 새로고침 없이 프론트엔드 단에서 실시간으로 계산 및 갱신됩니다.

----------------------------------------------------------------------
2. HTML 마크업 구조 (HTML Layout)
----------------------------------------------------------------------
전체 레이아웃은 Bootstrap v5 Grid 시스템(row, col-lg-8, col-lg-4)을 사용하여 
데스크톱과 모바일 환경에 모두 적응하는 반응형 레이아웃을 취하고 있습니다.

- [상단] 공통 헤더 및 내비게이션 바:
  - 다른 페이지와 동일한 SoolPair 로고 및 주 메뉴 탭 유지.
- [중앙 - 좌측 영역 (col-lg-8)] 기록 관리 보드:
  - 1단계 (도장 선택 영역): 위스키, 와인, 리큐르 버튼이 배치되어 클릭 시 active 상태로 변경.
  - 2단계 (달력 영역): 1일부터 31일까지 그리드 형태로 렌더링될 빈 컨테이너(`<div id="calendarGrid">`).
- [중앙 - 우측 영역 (col-lg-4)] 실시간 대시보드:
  - 주종 소비 비율 카드: 실시간으로 차트 세그먼트가 채워지는 가로형 프로그레스 바(`<div id="ratioBar">`)와 범례 목록(`<div id="ratioLegend">`).
  - 홈술 건강 관리 가이드 카드: 동적 알림 영역(`<div id="healthAlertArea">`)과 고정된 과음 위험성 주의 안내 문구 리스트.
- [하단] 공통 푸터 및 뉴스레터 구독 섹션:
  - SoolPair의 공통 푸터 정보 및 사용자가 지적했던 뉴스레터 구독(이메일 구독 신청 폼)을 복구하여 배치.

----------------------------------------------------------------------
3. CSS 스타일링 (CSS Stylesheet)
----------------------------------------------------------------------
Moderna 템플릿의 기본 테마를 깨뜨리지 않기 위해 스타일시트가 설계되었습니다.

- 도장 선택 버튼 (.stamp-item):
  - 둥근 점선 테두리 디자인으로 도장 이미지를 형상화.
  - 마우스 호버 시에는 테두리 색상만 자연스럽게 변경(transition: all 0.2s ease).
  - 활성화(.active) 시에는 위스키(노란색), 와인(붉은색), 리큐르(하늘색) 고유의 파스텔톤 배경과 
    진한 테두리 및 섀도우가 적용되어 어떤 도장이 선택되었는지 쉽게 구별할 수 있습니다.
- 달력 카드 (.day-card) 및 스탬핑 상태 (.stamped-*):
  - 7열 그리드 배치를 통해 달력 모양을 구성.
  - 기록되지 않았을 때는 일반 흰색 카드이나, 도장이 찍히면(`stamped`) 도장 유형 클래스
    (`.stamped-whiskey`, `.stamped-wine`, `.stamped-liqueur`)가 추가되면서 
    도장 이모지(🥃, 🍷, 🍹)가 튀어나오고(scale 1.2 회전 애니메이션) 카드의 배경색이 주종에 맞게 염색됩니다.
- 비율 차트 바 (.ratio-bar-wrapper):
  - flex 레이아웃을 활용하여 각 주종의 비율만큼 영역의 너비(`width`)가 동적으로 늘어납니다.

----------------------------------------------------------------------
4. JavaScript 비즈니스 로직 (JavaScript Logic)
----------------------------------------------------------------------
Vanilla JavaScript를 사용하여 상태 관리, DOM 조작, 동적 연산 기능이 유기적으로 연결되어 있습니다.

[A] 데이터 상태 및 모델 (Data State)
- `STAMP_TYPES`: 위스키, 와인, 리큐르에 대한 데이터 모델 사전 정의.
  (이름, 이모지, 기본 라벨 내용, CSS 클래스 등 포함)
- `selectedStampType`: 현재 마우스로 선택된 도장의 종류 ('whiskey', 'wine', 'liqueur' 중 하나).
- `homsoolLogs`: 사용자가 기록한 일별 데이터를 저장하는 Key-Value 형식 객체.
  - 구조 예시: `{ 1: { type: 'whiskey', detail: '짐빔 화이트 구매' }, 5: { type: 'wine', ... } }`
  - 사용자의 요청에 따라 기본 데이터는 비어있는 객체 `{}` 상태로 시작합니다.

[B] 주요 핵심 함수 (Key Functions)
1. `selectStamp(type)`
   - 사용자가 스탬프 툴바에서 다른 도장을 클릭할 때 트리거됩니다.
   - `selectedStampType`의 상태를 변경하고, UI상에서 다른 버튼들의 active 클래스를 제거하고 선택된 버튼만 강조합니다.

2. `initCalendar()`
   - 1일부터 31일까지 31번 루프를 돌면서 달력 카드의 기본 HTML 구조(일자 번호, 도장 컨테이너, 텍스트 라벨)를 생성하여 캘린더 그리드에 삽입합니다.
   - 각각의 카드에 클릭 이벤트 리스너를 바인딩하여 `handleDayClick(day)`를 호출하게 만듭니다.

3. `renderLogs()`
   - `homsoolLogs` 객체를 검사하여 이미 기록된 날짜 카드에 대해 알맞은 스탬프 클래스를 부착하고 이모지 및 라벨 텍스트를 채워 넣습니다.
   - 기록이 지워진 날짜 카드는 클래스를 초기화하고 내용을 숨깁니다.

4. `handleDayClick(day)`
   - 달력의 특정 일자를 클릭했을 때 실행되는 토글 함수입니다.
   - 만약 클릭한 일자에 이미 기록이 존재하면 `delete homsoolLogs[day]`를 통해 삭제합니다.
   - 기록이 없으면 현재 `selectedStampType` 상태를 기준으로 `homsoolLogs`에 새 객체를 할당합니다.
   - 데이터 조작 완료 후 `renderLogs()`와 `updateDashboard()`를 순차 호출해 화면을 실시간 갱신합니다.

5. `updateDashboard()`
   - 대시보드의 데이터를 갱신하는 핵심 집계 엔진입니다.
   - `homsoolLogs` 전체를 순회하며 위스키, 와인, 리큐르의 기록 횟수를 각각 누적하고 총 기록 횟수(`totalStamps`)를 구합니다.
   - **[핵심 기능 - 동적 과음 경고 분기]**:
     - `totalStamps`가 3회 이하: 녹색 테마의 "적정 상태 😊" 배지와 함께 권장 팁 노출.
     - `totalStamps`가 4회~8회: 노란색 테마의 "과음 주의 ⚠️"로 변하며 음주 조절 권장 가이드 노출.
     - `totalStamps`가 9회 이상: 빨간색 테마의 "과음 경고 🚨"로 변하며 간에 휴식기를 줄 것을 권고하는 문구 노출.
   - **[비율 차트 업데이트]**:
     - 각 주종 횟수를 총 횟수로 나누어 비율(%)을 소수점 반올림으로 구합니다.
     - 우측 대시보드의 `<div id="ratioBar">` 내부의 너비를 계산된 % 비율로 실시간 조정하고, 범례 리스트를 다시 그려 화면에 반영합니다.

----------------------------------------------------------------------
5. 코드 설명 브리핑 팁 (Presentation Tips)
----------------------------------------------------------------------
동료나 교수님에게 설명할 때 다음과 같은 순서로 진행하면 아키텍처의 완성도를 쉽게 인정받을 수 있습니다.

"SoolPair 홈술로그 시스템은 가벼운 홈술 다이어리 형태로, 데이터의 흐름(Data Flow)이 매우 명확합니다. 
데이터 상태인 'homsoolLogs'라는 객체가 싱글 소스 오브 트루스(Single Source of Truth) 역할을 담당하고 있으며, 
사용자가 달력을 클릭할 때마다 이 데이터가 토글 변경됩니다. 
데이터가 변경됨과 동시에 화면 렌더링 함수인 `renderLogs`와 실시간 통계 계산 함수인 `updateDashboard`가 
연쇄적으로 반응하여 화면의 프로그레스 비율 게이지와 3단계 건강 경고 알림 영역을 100% 프론트엔드 연산으로 
부드럽고 실시간으로 업데이트하는 reactive 형태의 UI 아키텍처입니다."
